# AGENTS — @to-much-talker/db

Drizzle ORM database package with dual-dialect schemas (SQLite + Postgres).

## Parallel-Schema Discipline (CRITICAL)

- Every table change must be reflected in BOTH `sqlite/schema.ts` AND `pg/schema.ts`
- Column types must be semantically equivalent:
  - `boolean` → SQLite: `integer({mode:'boolean'})`, Postgres: `boolean()`
  - `timestamp` → SQLite: `integer({mode:'timestamp'})`, Postgres: `timestamp()`
  - `json` → SQLite: `text({mode:'json'})`, Postgres: `jsonb()`
  - `bigint` → SQLite: `integer()` (no int64 in sqlite), Postgres: `bigint({mode:'number'})`
- NO dialect-specific features in schemas (no Postgres jsonb indexes, no SQLite rowid)
- Migrations are in `drizzle/sqlite/` and `drizzle/pg/` (generated by drizzle-kit in Task 9)

## The Discriminated Union Pattern (Task 8)

```typescript
type SqliteDb = { dialect: 'sqlite'; db: BetterSQLite3Database }
type PgDb = { dialect: 'pg'; db: PostgresJsDatabase }
type Db = SqliteDb | PgDb
```

Use `db.dialect === 'sqlite'` to narrow before calling dialect-specific methods.

## Import Pattern

```typescript
import { sqlite, pg } from '@to-much-talker/db'
// sqlite.guildSettings, pg.guildSettings
```

## Rules

- NO default exports — named exports only
- NO `as any` — use explicit types with `$type<T>()` helpers
- All TS imports MUST use `.js` extensions (NodeNext module resolution)
- Use `import type` for type-only imports (`verbatimModuleSyntax: true`)
- Both dialect schemas MUST stay in lockstep — adding a column to one without the other is a violation

## Discriminated Union Pattern

The `Db` type is a discriminated union:

```typescript
type SqliteDb = { dialect: 'sqlite'; db: ...; raw: Database; close(): void }
type PgDb = { dialect: 'pg'; db: ...; raw: postgres.Sql; close(): Promise<void> }
type Db = SqliteDb | PgDb
```

Always narrow before dialect-specific calls:

```typescript
if (isSqlite(db)) {
  db.raw.exec('PRAGMA journal_mode=WAL')
} else if (isPg(db)) {
  // postgres-js specific
}
```

## Opening a DB

```typescript
const result = await openDb(config.DATABASE_URL)
if (!result.ok) process.exit(1) // invalid URL scheme
const db = result.value
```
